How is the staging area different from the working directory and the repository? What value do you think it offers?

The staging area is the intermediate area in which files are stored before they're committed. It allows you to bring in a part of all the files in the working directory before you commit. 

How can you use the staging area to make sure you have one commit per logical change?

By making it a practice to only bring in files in the staging area that are related to one logical change

What are some situations when branches would be helpful in keeping your history organized? How would branches help?

A branch would be helpful when you're adding a big new feature and you are not sure if it would work out in the end. So you create a new branch for the feature instead of making changes to the master branch.
